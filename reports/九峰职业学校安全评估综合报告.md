## 3. 漏洞详情（按严重程度排序）

### 3.1 严重漏洞 🔴

#### 3.1.1 MQTT Broker未授权访问 (CVSS 9.8)

**设备：** 192.168.20.1 (锐捷网络网关)  
**端口：** 1883  
**版本：** mosquitto 1.5.9  
**运行时间：** 9,194,779秒（约106天）

**漏洞描述：**
MQTT Broker完全未授权，无需任何认证即可连接、订阅和发布消息。

**获取的敏感信息：**

| 信息类型 | 具体内容 |
|----------|----------|
| 网络拓扑 | 完整网络架构、设备位置、IP/MAC/SN |
| 设备信息 | 所有交换机、AP的完整标识信息 |
| 用户信息 | IP、MAC、主机名、设备型号、信号强度、流量统计 |
| 配置模块 | eweb_password, sonPassword, radius, aaa, wlanDot1x等 |
| 告警信息 | IP冲突检测、设备状态告警 |
| 网络ID | dev_C8:CD:55:F2:51:B2_1724495752 |

**MQTT统计信息：**
- 连接客户端：83个
- 订阅主题：1,145个
- 保留消息：50个
- 发送消息总数：227,003,084
- 接收消息总数：65,808,858

**泄露的配置模块：**
- `eweb_password`: Web管理密码配置
- `sonPassword`: SON密码配置
- `radius`: RADIUS认证配置
- `aaa`: AAA认证配置
- `wireless`: 无线配置
- `wlanDot1x`: 802.1X认证配置
- `timeReboot`: 定时重启配置
- `network_group`: 网络组配置

**攻击验证：**
```bash
# 订阅所有主题
mosquitto_sub -h 192.168.20.1 -p 1883 -t "#" -v

# 发布任意消息
mosquitto_pub -h 192.168.20.1 -p 1883 -t "test/security_check" -m "penetration_test" -r

# AP设备控制尝试
mosquitto_pub -h 192.168.20.1 -t "reyee/device/192.168.101.16/command" -m '{"action":"reboot","reason":"test"}' -r

# 批量配置修改尝试
mosquitto_pub -h 192.168.20.1 -t "reyee/device/all/command" -m '{"action":"config_update","ssid":"HACKED"}' -r
```

**结论：** 消息成功发布，无任何权限校验。可以执行设备控制命令，120秒内收集497条消息。

### E.4 透明代理指纹再确认

在对光猫进行 HTTP 探测时，发现一个**关键证据**：

```http
HTTP/1.1 200 OK
Content-Length: 444
Proxy-Connection: keep-alive
X-Frame-Options: SAMEORIGIN
```

**`Proxy-Connection: keep-alive` 出现在 192.168.1.1 的响应中。**

进一步对比测试：

| 目标 | Proxy-Connection 头 | 分析 |
|------|---------------------|------|
| 192.168.1.1（光猫） | ✅ 存在 | 内网到光猫的流量也被代理 |
| 192.168.20.1（锐捷网关） | ✅ 存在 | 网关自身响应也被代理 |
| www.baidu.com | ✅ 存在 | 外网流量被代理 |

**结论：** 该 `Proxy-Connection` 头**不是光猫自身插入的**，而是位于本机与光猫之间的**透明代理设备**统一添加的。结合网络拓扑，最可能的代理设备是 **192.168.20.1 锐捷网关**，它对所有经过的 HTTP 流量（无论内网还是外网）都做了一层代理转发。

这意味着：
1. 学校网络的所有 HTTP 明文流量**全部可被该设备读取和记录**
2. 该设备有能力在 HTTP 层进行**内容过滤、广告注入、URL 重定向**
3. 如果该设备被攻破，攻击者可获得全校师生的**完整上网行为日志**
